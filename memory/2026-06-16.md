## Ключевые решения (зафиксированы, не переспрашивать)
- Формат: один `index.html`, офлайн, без библиотек, мобайл-первый.
- Уровень — средний. Тренировки Пн/Ср/Пт, ~45–60 мин.
- Гири 2×16 кг → прогрессия повторами и усложнением варианта, НЕ весом.
- «Сегодня» — авто по календарю (дата старта + дни Пн/Ср/Пт), хранится в localStorage.
- Главный экран = сегодня + отдельный обзор года (52 недели).
- НЕ делаем без запроса: галочки выполнения, запись весов, графики, аккаунты, сервер.

## Как устроен index.html (для быстрого входа завтра)
- Данные программы в JS: `DAYS` (A/B/C, у каждого упражнения 4 варианта по блокам + tip `Техника`), `SCHEMES`, `BLOCKS`, `WARMUP`, `COOLDOWN`.
- Движок: `currentSession()` → {idx, isRest, refDate}; `decode(idx)` → {year, week, dayIndex}; `buildWorkout(week, dayIndex)`.
- Год = 4 блока ×12 недель + разгрузка/тест (49–52); каждая 4-я неделя блока = разгрузка (2 подхода).
- Режим просмотра: `browseWeek`/`browseDay` + панель «← К сегодня».

## Логика дня отдыха (итог последней правки)
- В день отдыха показывается **экран отдыха**, тренировка скрыта.
- Кнопка **«Вернуться к прошлой тренировке»** (`peekPrev()`) — открывает прошлую тренировку в просмотре (для догона пропуска). График не трогает.
- Кнопки **«− сдвинуть назад / + вперёд»** в Настройках — двигают весь график (offset в localStorage). Нужны, только если расписание реально разъехалось.
